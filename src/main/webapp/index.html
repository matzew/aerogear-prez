Les Pipes pour iOS


 NSURL* projectsURL = [NSURL URLWithString:@"http://www.server.com"];
 AGPipeline *server = [AGPipeline pipelineWithBaseURL:projectsURL];

 // create the 'Pipe' that will point to the remote '/developer/' endpoint
 id<AGPipe> developerPipe = [server pipe:^(id<AGPipeConfig> config) {
   [config setName:@"/team/developers"];
 }];

   _data= responseObject;
   [self.tableView reloadData];
 } failure:^(NSError *error) {
   NSLog(@"An error has occured during pipe read! \n%@", error);

Les Stores pour iOS


  NSString* type = @"SQLITE"; //@"PLIST";
  AGDataManager* dm = [AGDataManager manager];
  // Add a new (default) store object:
  _store = [dm store:^(id config) {
    [config setName:@"recipes"];
    [config setType:type];
  }];

  _recipes = [[NSMutableArray alloc] init];
  NSArray* myRecipes = [_store readAll];
  for (id item in myRecipes) {
    AGRecipe *rec = [[AGRecipe alloc] initWithDictionary:item];
    [_recipes addObject:rec];
  }

3: Les fonctionalités cross-platform


Unified Push Server

Push Notifications

  • Small messages for notifications!
  • No custom connection to maintain:
    • Device "phones" home...
    • Multiplexing over one connection to the actual Push Network
  • Platform specific APIs to read the notification!

Apple Push Notification Service

  • iOS and Safari
  • Production/Development environment
  • Complex setup
    • Provisioning Profile(s)
    • SSL Certificate(s)
  • Device identified by deviceToken
  • very simple client APIs

Google Cloud Messaging

  • (Google) Android and Chrome
  • Simple setup (SenderID + Google API Key)
  • Device identified by registrationID
  • complex handling on Android:
    • lot's of XML (Manifest) to write
    • lot's of Android code to implement

Mozilla SimplePush

  • FirefoxOS and Firefox Desktop
  • Signaling/notification system (not message carrying)
  • PUB / SUB
  • Subscriptions identified by channelID
  • Mozilla "spec" due to W3C’s Push-API issues
    • Protocol defined (Ref. Server: written in Go)
    • JavaScript Client API defined
  • AeroGear Quickstart:
  • https://github.com/aerogear/aerogear-simplepush-quickstart/
